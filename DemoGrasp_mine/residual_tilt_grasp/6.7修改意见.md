# Tilt-Aware Residual RL 重构方案（V2）

## 1. 研究目标

在不破坏已有 0° 抓取策略的前提下，让策略适应倾斜桌面上的抓取。

当前实验已经证明固定 Canonical Trajectory 具有一定泛化能力：

| 倾角 | 未进行残差学习时的成功率（约） |
|---|---:|
| 0° | 0.82 |
| 15° | 0.61 |
| 30° | 0.40 |
| 45° | 0.28 |

因此当前核心问题不是参考轨迹完全失效，而是：

1. 倾斜抓取主要需要修改手腕位姿还是手指姿态？
2. 合理的残差范围是多少？
3. 如何在学习新角度时严格保护旧策略？
4. PPO 是否能够稳定找到有效残差？

---

## 2. 对现有方案的判断

### 2.1 成立的部分

- 将动作扩展为手腕残差和手指残差。
- 对所有残差设置明确范围。
- 冻结已有策略，避免灾难性遗忘。
- 每个倾角独立归一化 Advantage。
- 进行 Wrist-only、Finger-only、Hybrid 消融。
- 先使用 Best-of-N 判断动作空间是否存在有效解。
- 第一阶段保持现有 Observation，不同时重构所有模块。

### 2.2 必须修正的认识

当前旧 PPO 已经输出 18 维手指偏移，并不是单纯执行 Demo 手势。

旧策略实际形成：

```text
q_base = q_demo + old_policy(observation)
```

新的残差策略应建立在冻结旧策略之上：

```text
q_final = q_base + delta_q_residual
```

不能重新训练一个网络直接替代旧 PPO，否则仍然会破坏已有能力。

---

## 3. 总体数据流

```text
                       +-> Frozen Baseline Policy -> 18D base finger action
Observation -----------+
                       +-> Residual Policy --------> 6D wrist + 18D finger residual

Canonical wrist trajectory + bounded wrist residual -> modified wrist trajectory
Demo grasp + baseline action + bounded finger residual -> modified hand trajectory
```

### 3.1 Baseline Policy

- 使用已有 `model_3200.pt` 或指定 checkpoint。
- 网络参数完全冻结。
- 推理使用确定性均值动作，不使用探索噪声。
- 不进入 residual PPO optimizer。
- 保存残差 checkpoint 时记录 baseline checkpoint 路径和校验信息。

### 3.2 Residual Policy

- PPO 只优化残差策略。
- 输出维度由 `residual_mode` 决定：

```text
wrist  -> 6
finger -> 18
hybrid -> 24
```

- PPO 的 log probability、entropy、KL 和 Advantage 只针对残差动作。

## 5. Residual Action Space

## 5.1 Wrist Residual

网络输出：

```text
a_wrist = [atx, aty, atz, arx, ary, arz]
```

经过 `tanh` 和尺度限制：

```text
delta_p_table = translation_scale * tanh(a_translation)
delta_omega_table = rotation_scale * tanh(a_rotation)
```

默认范围：

```text
translation_scale = [0.02, 0.02, 0.02] m
rotation_scale = [10, 10, 10] degree
```

旋转部分解释为 rotation vector / axis-angle：

```text
delta_R = Exp(delta_omega^)
```

不使用 roll/pitch/yaw 依次组合，避免欧拉角顺序和奇异性问题。

### 5.1.1 残差坐标系

手腕残差定义在桌面坐标系：

```text
delta_p_world = R_table * delta_p_table
delta_R_world = R_table * delta_R_table * inverse(R_table)
```

### 5.1.2 位姿复合方式

第一版不要把 `delta_T` 围绕世界原点或桌面原点左乘整条轨迹，否则旋转残差会产生额外的大范围位置摆动。

建议对每个参考位姿使用：

```text
p_ref'(t) = p_ref(t) + delta_p_world
R_ref'(t) = delta_R_world * R_ref(t)
```

即：

- 平移是统一偏移。
- 旋转修改手腕朝向。
- 不因旋转而绕桌面中心改变手腕位置。

保留以下接口，供后续实验：

```text
+residual_pose_composition=orientation_only
+residual_pose_composition=se3_left
+residual_pose_composition=se3_local
```

第一版默认 `orientation_only`，含义是上述“位置相加、朝向左乘”方式。

### 5.1.3 残差作用范围

第一版使用同一个固定残差修正整个 reaching 和 tracking 轨迹：

```text
+residual_scope=full_trajectory
```

这样不会在抓取关键帧发生位姿跳变。

后续可消融：

```text
+residual_scope=pregrasp_and_grasp
+residual_scope=grasp_and_lift
+residual_scope=grasp_keyframe
```

若只修正部分轨迹，必须使用平滑权重进行淡入/淡出，禁止突然切换。

## 5.2 Finger Residual

冻结旧策略首先产生：

```text
q_base = q_demo + baseline_action
```

残差策略再产生：

```text
q_final = q_base + delta_q
```

每个关节的尺度由 Demo 闭合量和关节范围共同决定：

```text
close_j = abs(q_grasp_demo_j - q_pre_demo_j)
joint_range_j = q_upper_j - q_lower_j

lambda_q_j = clamp(
    alpha * close_j,
    min_ratio * joint_range_j,
    max_ratio * joint_range_j
)

delta_q_j = modulation(theta) * lambda_q_j * tanh(a_j)
```

默认建议：

```text
alpha = 0.1
min_ratio = 0.01
max_ratio = 0.15
```

最终结果必须再次按照 URDF joint limits clamp。

## 6. 倾角调制与 0°保护

默认使用归一化正弦调制：

```text
s(theta) = clamp(
    sin(abs(theta)) / sin(theta_max),
    0,
    1
)
```

其中 `theta_max` 默认取训练角度绝对值的最大值，也允许手动指定。

性质：

```text
theta = 0       -> s = 0
theta = theta_max -> s = 1
```

第一阶段 Wrist 和 Finger 都进行调制：

```text
delta_wrist = s(theta) * bounded_wrist_action
delta_finger = s(theta) * bounded_finger_action
```

因此 0°严格满足：

```text
final_policy(0deg) == frozen_baseline_policy(0deg)
```

提供接口：

```text
+tilt_modulation=normalized_sin
+tilt_modulation=sin
+tilt_modulation=linear
+tilt_modulation=none

+wrist_tilt_modulation=true
+finger_tilt_modulation=true
+tilt_modulation_max_angle=45
```

注意：若 `finger_tilt_modulation=false`，0°不再受到严格保护，只能用于后续消融。

## 7. Per-Angle Advantage Normalization

当前每个 rollout 对应一个 sim 级别倾角。平衡训练时：

```text
rollout 0 -> 0°
rollout 1 -> 15°
rollout 2 -> 30°
rollout 3 -> 45°
```

Rollout Storage 必须额外保存每条 transition 的 `angle_id`。

原始 Advantage：

```text
A = reward - value
```

对每个角度分别归一化：

```text
A_theta = (A_theta - mean(A_theta)) / (std(A_theta) + epsilon)
```

然后再将所有角度样本混合并随机生成 minibatch。

接口：

```text
+per_angle_advantage=true
+advantage_eps=1e-8
+advantage_min_samples=32
```

单角度训练时该逻辑自然退化为普通全局归一化。

## 8. Residual Capacity Study（P0，先于 PPO）

目的：先判断动作空间是否存在明显更优解。

固定：

- baseline checkpoint
- 物体初始状态
- 参考轨迹
- 不更新任何网络

对每个场景随机采样 N 组有界残差，执行完整 Move-Close-Lift，然后统计：

```text
baseline success
mean random success
best-of-N success
oracle improvement = best-of-N - baseline
```

实验模式：

```text
A: finger-only
B: wrist-only
C: hybrid
```

建议首先测试：

```text
angles = [15, 30, 45]
N = [8, 16, 32]
```

必须保证 Best-of-N 的 N 个候选使用相同物体、相同初始位姿，否则不能说明残差搜索带来的提升。

解释：

- Wrist-only 明显提升：倾角适应主要依赖位姿修正。
- Finger-only 明显提升：手势修正仍然重要。
- Hybrid 远高于两者：两类残差存在互补。
- 三者均无提升：范围太小、作用方式错误，或固定轨迹能力不足。

建议新增独立脚本：

```text
residual_tilt_grasp/residual_capacity_study.py
```

该脚本不依赖 PPO。

## 9. PPO 训练阶段

容量实验确认动作空间有效后，再实现 residual PPO。

### 9.1 初始训练顺序

1. 15° Wrist-only。
2. 15° Finger-only。
3. 15° Hybrid。
4. 在最佳模式上测试 30°。
5. 测试 45°。
6. 最后进行 `[0,15,30,45]` 平衡训练。

15°实验目标应首先超过冻结 baseline 的约 0.61。

### 9.2 训练约束

- baseline policy 永久冻结。
- residual policy 单独 optimizer。
- PointNet 是否共享需要显式决定。
- 第一版建议 residual policy 使用独立 PointNet，避免 residual critic 修改 baseline 特征。
- 0°残差经过调制后严格为零。
- minibatch 随机混合角度。
- Advantage 按角度归一化。

### 9.3 PPO 超参数接口

```text
+residual_action_std=0.1
+residual_learning_rate=1e-4
+residual_epochs=2
+residual_cliprange=0.1
+residual_entropy_coef=0.0
+residual_value_loss_coef=1.0
+residual_max_grad_norm=1.0
+residual_sampler=random
```

这些值只作为默认建议，必须保存在每个 run 的 config 中。

## 15. 最终假设

> Canonical Trajectory 提供抓取结构先验；  
> Frozen Baseline Policy 提供已学习的手指抓取能力；  
> Residual RL 只学习倾角导致的有限位姿与手势补偿；  
> Diffusion / Flow Matching 后续负责生成物体与任务条件下的抓取模式。

当前第一优先级不是立即训练完整 residual PPO，而是先通过 Residual Capacity Study 证明动作空间确实包含更优解。

## 16. 训练停滞诊断与关键修正（2026-06-07）

### 16.1 问题表现

第一版 Residual PPO 能严格复现冻结 Baseline，但训练约 30 个 iteration
后成功率几乎不变：

```text
0° 约 0.86
15° 约 0.64
30° 约 0.43
45° 约 0.30
```

这说明 0°保护和残差组合逻辑正确，但 Residual Policy 没有有效离开零残差
附近。此时继续增加 iteration 不能解决根本问题。

### 16.2 排除“学习率太低”

停滞版本的学习率一度达到 `5e-4`，已经高于旧策略微调使用的
`3e-5`。因此问题不是简单的更新步长不足。盲目继续提高学习率可能只会
放大噪声和破坏 PPO 稳定性。

排查中还通过人工构造的二值奖励完成了最小梯度测试：

- residual actor 输出层梯度非零；
- optimizer step 后输出层权重发生变化；
- PPO 的 action、log-prob、advantage 和 backward 链路没有断开。

因此，问题属于“学习信号和探索尺度不足”，而不是“代码完全没有更新”。

### 16.3 根因一：物理探索尺度过小

旧策略的动作直接乘：

```text
randomizeGraspPoseRange = 1
```

当旧策略 `action_std=0.15` 时，典型手指探索量约为 `0.15 rad`。

第一版 residual 的手指尺度为：

```text
lambda_q = 0.1 * |q_grasp_demo - q_pre_demo|
residual_action_std = 0.1
```

Shadow Hand 示范闭合量平均约为 `0.35 rad`，所以平均物理探索量只有：

```text
0.1 * 0.35 * 0.1 = 0.0035 rad
```

它比旧策略的典型探索小几十倍。二值奖励要求动作真正跨过成功判定边界，
毫米级或几千分之一弧度的扰动通常无法产生足够的成功/失败对比。

需要始终区分：

```text
网络动作 std != 最终物理动作 std
最终物理探索 = 网络 std * residual scale * tilt modulation
```

后续设计探索参数时，必须报告最终的米、度和弧度，而不能只看归一化
动作空间中的 `action_std`。

### 16.4 根因二：Residual MLP 从随机特征开始

第一版只把 Baseline PointNet 权重复制给 Residual Policy，而 actor MLP 和
critic 仍为随机初始化。虽然最终输出层为零能保护初始行为，但稀疏二值奖励
需要从随机表征重新学习“哪些物体状态对应哪些残差”，优化难度过高。

修正后的初始化方式：

1. 从 Baseline 复制 PointNet。
2. 从 Baseline 复制 actor 的所有共享特征层。
3. 从 Baseline 复制 critic。
4. 对新增的 3 维 `worldtilt` 输入权重初始化为零。
5. Residual 最终输出层继续严格置零。
6. Residual 的 `log_std` 不从 Baseline 复制，使用独立探索配置。

这样同时满足：

```text
初始 deterministic residual = 0
初始行为 = Frozen Baseline
Residual Policy 继承已有物体和抓取特征
```

### 16.5 修正后的默认训练参数

```text
residual_action_std=0.3
finger_residual_alpha=0.3
finger_residual_min_ratio=0.02
finger_residual_max_ratio=0.3
residual_learning_rate=3e-4
residual_entropy_coef=1e-3
residual_initialize_from_baseline=true
residual_freeze_backbone=true
```

含义：

- 增大初期探索，使动作能够跨过二值成功边界。
- 通过少量 entropy bonus 防止探索方差过快塌缩。
- 学习率保持在正常 PPO 范围，而不是依靠极端学习率弥补探索不足。
- 冻结继承的 PointNet，保护已有点云表示并减少显存。

### 16.6 显存修正

多角度平衡训练中，每次 update 的样本数为：

```text
num_envs * num_angles
```

例如 `7000 env × 4 angles = 28000` 条。若仍使用 4 个 minibatch，则单次
backward 包含 7000 条点云，容易爆显存。

当前默认按 rollout 数同步增加 minibatch：

```text
nminibatches = original_nminibatches * nsteps
```

四角度时为 16 个 minibatch，每批仍约 1750 条。完整 28000 条数据都会
参与 PPO，只是分批 backward，降低峰值显存。

### 16.7 新增诊断指标

每轮记录：

```text
sampled   # 含探索噪声的归一化残差绝对值
mean      # deterministic policy mean，表示真正学到的残差
head_norm # residual 输出层权重范数
wrist     # 最终物理手腕平移，单位 m
finger    # 最终物理手指残差，单位 rad
```

判断方式：

- `sampled > 0`、`mean ≈ 0`：只有随机探索，策略尚未学到方向。
- `head_norm` 不变：输出层基本没有得到有效更新。
- `mean` 和 `head_norm` 增长：策略确实离开零残差。
- 物理 `wrist/finger` 很小：尺度或倾角调制仍限制了有效探索。
- 残差明显增大但成功率不升：需要检查动作方向、容量或奖励设计。

### 16.8 实验结果

修正后单独训练 15°：

```text
Balanced tilt success: 0.882
15deg=0.882
Residual magnitude:
sampled=0.3128
mean=0.2197
head_norm=0.4684
wrist=0.0170m
finger=0.0289rad
```

相比冻结 Baseline 的 15°约 `0.64`，成功率提升至 `0.88`。策略均值和输出
层范数均明显非零，证明提升来自学到的确定性残差，而不只是采样波动。

在 `[0°, 15°]` 联合训练中：

```text

0deg=0.854
15deg=0.846
Residual magnitude:
sampled=0.2780
mean=0.1647
head_norm=0.3393
wrist=0.0071m

```

结果说明：

- 0°能力仍保持在 Baseline 附近。
- 15°显著超过未训练 Baseline。
- 倾角调制和 Frozen Baseline 成功保护了原能力。
- Residual RL 的动作空间和总体结构是有效的。

### 16.9 对后续修改的启发

1. 不要只根据归一化 `action_std` 判断探索大小，必须换算为实际物理量。
2. 强 Baseline 场景下，Residual Policy 应继承 Baseline 表征，而不是从随机
   MLP 开始。
3. 零输出初始化只负责保护初始行为，不能替代有效特征初始化和充分探索。
4. 成功率停滞时先检查 `mean`、`head_norm` 和物理残差，再决定是否修改奖励。
5. 学习率不是万能解释。应先排除梯度断链、探索不足和尺度错误。
6. 多任务训练必须同步调整 minibatch 数，避免样本总量增加后改变单次
   backward 的规模。
7. 单角度实验仍然是最有效的诊断手段。先证明 15°能够超过 Baseline，再扩展
   到 30°、45°和多角度联合训练。
8. 后续 dense reward 或 curriculum 应建立在当前已验证的残差结构上，不应
   再同时替换 Baseline、动作定义和 observation。
