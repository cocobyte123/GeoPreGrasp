# Object-Centric Pregrasp 实现说明

本文档记录当前预抓取模块的核心思想、数据定义和代码工程实现，方便后续整理成论文方法部分。

当前这套实现的主线是：

```text
不重新训练 PPO 主策略，而是在 PPO 执行前，为每个物体选择一个更合适的预抓取初始化姿态。
```

旧版本的 M 点热图是把手的位置离散到空间点上，然后从热图里选空间 bin。现在的版本不再直接学习空间点，而是学习一个更贴近 PPO 初始动作分布的参数化预抓取先验：

```text
P(success | object geometry, object pose, relative yaw, tilt, pitch)
```

其中 `relative yaw / tilt / pitch` 是预抓取候选角度。学习器只负责给候选角度打分，真正执行抓取的还是原来的 residual PPO 策略。

## 1. 核心问题

原始 PPO 在训练时已经学会了从某个初始化手势分布出发抓取物体。这个分布本质上不是完整的 3D 空间分布，而是围绕物体的一组有限接近方向。我们希望扩展这个预抓取初始化分布，让它根据物体几何选择更合适的接近方向。

直接把手放到任意球壳位置会导致两个问题：

1. 很多初始化姿态落到 PPO 没见过的输入域，策略会退化。
2. 空间热图离散化有损，不能很好表达“相对物体几何的接近方向”。

因此现在采用参数化角度候选：

```text
candidate = (relative_yaw, tilt, pitch)
```

这三个角度不是让 PPO 直接学习新动作，而是改变 PPO 执行前的真实初始手势。

## 2. 角度定义

### 2.1 reference yaw

`reference_yaw` 是物体在桌面平面上的参考朝向。当前稳定版本使用：

```text
reference_yaw = pca_short(object point cloud projected to table xy)
```

也就是把物体点云投影到桌面 `xy` 平面，对投影点做 PCA，取短轴方向作为参考方向。

选择短轴的原因是：很多物体适合从窄边或短边方向接近，例如盒子、瓶子、块状物。这样 `relative_yaw=0` 的语义就变成“沿几何短轴定义的参考方向接近”，而不是世界坐标里的绝对 0 度。

代码位置：

```text
PregraspPrior/src/oc_pregrasp/data/features.py
DemoGrasp_mine/build_Pregrasp_data/yaw_pregrasp/collect/collect_yaw_pregrasp_rollouts.py
DemoGrasp_mine/new_eval_pregraspRL/eval_yaw_scorer_SE_residual.py
```

当前支持的 yaw frame：

```text
absolute   world yaw = candidate yaw
object     world yaw = object yaw + candidate yaw
pca_long   world yaw = object projected PCA long axis + candidate yaw
pca_short  world yaw = object projected PCA short axis + candidate yaw
```

当前默认使用：

```text
pregrasp_yaw_frame=pca_short
```

### 2.2 real yaw

学习器预测的是相对角度 `relative_yaw`，仿真真正执行的是：

```text
real_yaw = reference_yaw + relative_yaw
```

代码中对应字段：

```text
yaw_deg             relative yaw
reference_yaw_deg   object-centric reference yaw
world_yaw_deg       real execution yaw
```

### 2.3 tilt 和 pitch

`tilt` 和 `pitch` 复用原来 SE residual grasp 里的手势扩展方式。当前常用候选集合是：

```text
yaw:   0, 30, ..., 330
tilt:  0, 15, 30
pitch: 0, 15, 30
```

总候选数：

```text
12 * 3 * 3 = 108
```

最小验证集 `min10` 下，10 个物体一批正好是：

```text
10 objects * 108 candidates = 1080 envs
```

因此 min10 采集脚本默认使用 `num_envs=1080`。

## 3. PPO 输入归一化

这里的关键点是：真实仿真里的手会按照 `real_yaw` 旋转，但 PPO 看到的输入要尽量保持在训练分布里。

实现方式是对 PPO observation 做 scene-level yaw 归一化：

```text
真实执行：手和轨迹按 real_yaw 放到世界里
PPO 输入：把观察到的手、物体、点云绕物体中心旋回 -real_yaw
```

也就是说，仿真世界里手确实在不同 yaw 方向；但是送给 PPO 的 `eefpose / objpose / objinitpose / objxyz / objpcl` 等世界坐标量会被转换到归一化坐标中。

这样做的效果是：

```text
PPO 仍然处理接近原始训练分布的输入；
外部预抓取模块负责扩展真实世界中的接近方向。
```

当前稳定配置：

```text
pregrasp_yaw_canonical_obs=True
pregrasp_yaw_canonical_obs_scope=scene
pregrasp_yaw_direct_base_qpos=True
```

其中：

```text
canonical_obs=True
```

表示启用 yaw 归一化观察。

```text
canonical_obs_scope=scene
```

表示不仅归一化手，也归一化物体位姿和物体点云。这一点很重要，因为 PPO 输入里包含 `objinitpose` 和 `objpcl`。如果只转手不转物体，手和物体的相对关系会在 observation 里不一致。

```text
direct_base_qpos=True
```

表示虚拟手根部姿态直接写入 qpos，避免 Euler 角分支跳变导致手在可视化中突然自转。


采集过程：

1. IsaacGym 创建多个并行环境。
2. 每个 env 选择一个物体和一个候选角度 `(yaw, tilt, pitch)`。
3. 根据物体点云和物体姿态计算 `reference_yaw`。
4. 计算真实执行角 `world_yaw = reference_yaw + yaw`。
5. 用真实角度初始化手。
6. PPO 接收 scene-canonical observation 执行抓取。
7. episode 结束后记录 success / table hit / clearance 等标签。

每个 object 每个 batch 保存一个压缩 npz：

```text

```

```text
hit_table
min_clearance
real_palm_pose_world
canonical_palm_pose_world
object_point_cloud_object
```

其中：


是真实仿真世界里的初始手掌位姿。

```text

```

是把真实手掌位姿旋回 canonical frame 后的位姿，用来检查归一化是否正确。

## 6. 学习器输入与模型

当前第一个学习 baseline 是 PointNet-style scorer：

```text
PregraspPrior/src/oc_pregrasp/models/scorer_mlp.py
```

模型形式：

```text
object point cloud -> per-point MLP -> max pool -> object feature
candidate angle -> sin/cos encoding
object pose / geometry feature -> auxiliary feature
concat -> MLP -> success logit
```

前向输入：

```text
point_cloud:  [B, N, 3]
angle:        [B, 6]    # sin/cos(yaw, tilt, pitch)
object_pose:  [B, 7] or [B, 7 geometry features]
```

输出：

```text
logit = scorer(point_cloud, angle, object_pose)
score = sigmoid(logit)
```

训练目标是二分类：

```text
label = success in rollout
loss = BCEWithLogitsLoss(logit, label)
```

这里的“分类准确率”不是最终目标本身。它只是衡量单个候选是否被预测为成功/失败。论文里更重要的指标应该是：

```text
Top1 by scorer: 对每个物体选择 scorer 分数最高的候选，看真实成功率
Oracle top1:    对每个物体选择 rollout 中真实成功率最高的候选，上限
Uniform:        随机选候选的平均成功率
Base000:        固定 yaw=0, tilt=0, pitch=0 的成功率
Gym rollout:    scorer 选择候选后，在 IsaacGym 中真实跑 PPO 的成功率
```

## 7. pca_short feature mode

当前更稳定的学习输入是：

```text
feature_mode=pca_short
```

代码：

```text
PregraspPrior/src/oc_pregrasp/data/features.py
PregraspPrior/src/oc_pregrasp/data/rollout_dataset.py
```

处理方式：

1. 从 npz 中读取 `object_point_cloud_object`，这是物体自身坐标系下的点云。
2. 使用 `object_pose_world` 把点云旋到世界相对方向。
3. 在桌面 xy 平面计算 PCA 短轴 `reference_yaw`。
4. 把点云中心化，并旋转 `-reference_yaw`。
5. 额外生成 7 维几何/姿态特征：

```text
short_extent
long_extent
height
anisotropy = (long_extent - short_extent) / (long_extent + short_extent)
object_z
sin(object_yaw - reference_yaw)
cos(object_yaw - reference_yaw)
```

这样做的目的：

```text
让学习器看到的点云方向和 yaw 标签定义对齐，减少绝对坐标朝向带来的干扰。
```

## 8. 训练

训练入口：

```text
PregraspPrior/src/oc_pregrasp/train/train_scorer.py
```

常用脚本：

```text
PregraspPrior/scripts/train_scorer_mlp_geom_yaw12_tilt3_pitch3.sh
PregraspPrior/scripts/train_scorer_mlp_geom_yaw12_tilt3_pitch3_min10.sh
```

训练逻辑：

1. `YawPregraspRolloutDataset` 遍历所有 `batch_*.npz`。
2. 每一条样本对应一个候选角度的 success label。
3. 训练集和验证集按样本随机划分，默认 `val_fraction=0.15`。
4. 使用 `AdamW` 和 `BCEWithLogitsLoss`。
5. 每个 epoch 保存 `last.pt`，验证准确率更高时保存 `best.pt`。

当前 checkpoint 中会保存：

```text
model
optimizer
feature_mode
epoch
args
val
```

因此评估脚本可以自动读取 checkpoint 的 `feature_mode`。

离线 top1 只能说明标签数据上有效，最终还要在 IsaacGym 中真实执行 PPO。

在线评估入口：

```text

```

```text

PregraspPrior/scripts/compare_base_vs_scorer_geom_gym_pca_short_yaw12_tilt3_pitch3_scene_min10.sh
```

在线评估流程：

1. 当前 env reset 后得到物体点云和物体初始位姿。
2. 构造所有候选 `(yaw, tilt, pitch)`。
3. 用 scorer 对所有候选打分。
4. 根据采样模式选择候选：

```text

scorer_topk    在 top-k 中选择
scorer_sample  按分数采样
```

5. 按选中的候选设置真实初始手势。
6. PPO 在 canonical observation 下执行。
7. 输出真实 rollout success。

对比模式支持在同一次运行里比较：

```text
base   固定或随机使用原始角度策略
scorer 使用学习器选择角度
```

关键参数：


## 12. 与论文方法部分的对应关系

论文里可以把这一部分概括为一个 object-centric pregrasp prior：

```text
给定物体几何和当前物体姿态，预测一组参数化预抓取候选的成功概率；
选择最高概率候选作为 PPO 的初始化；
PPO 策略本身不改变，只改变执行前的预抓取姿态。
```

方法可以拆成三层：

### 12.1 Object-centric candidate parameterization

用物体桌面投影的 PCA 短轴定义参考 yaw：

```text
r(o, q) = yaw(PCA_short(project_xy(R(q) * P_object)))
```

候选角：

```text
a = (delta_yaw, tilt, pitch)
```

真实执行角：

```text
psi = r(o, q) + delta_yaw
```

### 12.2 Policy-preserving canonical execution

真实手势在世界中绕物体旋转，但 PPO observation 被旋回 canonical frame：

```text
obs_canonical = T(-psi, object_center) * obs_world
```

这样可以扩展真实初始化，同时保持 PPO 输入域接近训练分布。

### 12.3 Learned pregrasp scorer

从仿真 rollout 得到监督标签：

```text
y = 1 if PPO succeeds from candidate a else 0
```

训练 scorer：

```text
s_phi(P_object, q_object, a) ~= P(y=1 | P_object, q_object, a)
```

部署时：

```text
a* = argmax_a s_phi(P_object, q_object, a)
```

然后用 `a*` 初始化手，再由 PPO 执行闭环抓取。

## 13. 当前结论

目前 min10 离线结果已经验证了核心逻辑：

```text
uniform 约 0.76
base000 约 0.85
MLP top1 约 0.915
oracle 约 0.955
```

这说明：

1. 角度候选空间里确实存在比默认初始化更好的预抓取姿态。
2. 这些选择不是随机噪声，PointNet scorer 可以从物体几何中学到。
3. 预抓取模块可以作为 PPO 前面的独立初始化选择器，不需要改 PPO 本体。

后续论文里更有说服力的实验应包括：

```text
Base PPO / uniform candidates / oracle candidates / learned scorer
```

并分别报告：

```text
offline top1 label success
IsaacGym real rollout success
angle distribution
per-object success gain
```
